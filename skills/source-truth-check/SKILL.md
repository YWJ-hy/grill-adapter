# source-truth-check (Verify)

This is the host-agnostic **source-of-truth Verify** touchpoint. Some repo paths are *derived from an upstream source of truth* (generated clients, vendored contracts, OpenAPI specs). Editing them directly instead of regenerating them silently diverges the repo from its real source. This skill renders the project's configured policy as a bounded prompt and uses it to check a draft spec or plan for such conflicts — the same way `wiki-research` discloses wiki constraints, so the check is deterministic rather than relying on prose reminders.

It does not patch any host skill. A host wires it in by convention:

- **grill**: run `/source-truth-check` during `/to-spec` (render `spec-pre`) and `/to-tickets` (render `plan-pre`, and `plan-review` when reviewing the plan). `/to-spec` and `/to-tickets` are never patched.
- **plain Claude Code**: run `/source-truth-check` yourself before finalizing a spec or plan.

Policy is root-specific: project `sourceOfTruth` lives in `.adapter/settings.json`; a shared root uses `.shared-adapter/settings.json`. The policy is a **prompt guard only** — do not add a mandatory source-of-truth section to the spec/plan, run a semantic verifier agent, or create source-of-truth sidecar artifacts.

## Settings shape

```jsonc
{
  "sourceOfTruth": {
    "heuristics": false,
    "sources": [
      { "paths": ["src/services/generated/**"], "role": "truth", "edit": "never" },
      { "paths": ["openapi/**", "!openapi/draft/**"], "role": "truth", "edit": "ask" },
      { "paths": ["**/*.mock.ts", "**/*.fixture.ts"], "role": "evidence" },
      { "paths": ["dist/**", "node_modules/**"], "role": "ignore" }
    ]
  }
}
```

- `role: truth` + `edit: never` — never edit directly; regenerate from the upstream source.
- `role: truth` + `edit: ask` — edit only after explicit user authorization.
- `role: evidence` — informational; lint findings are warnings only.
- `role: ignore` — excluded from classification.
- `!pattern` negates a glob.

## Verify at spec time

```bash
python3 __GRILL_ADAPTER_ROOT__/scripts/source_truth_settings.py <repo-root> --render-prompt spec-pre
```

If stdout is non-empty, include it as a short `Source-of-Truth Policy` prompt input while drafting the spec: do not spec direct edits to `truth/edit: never` paths, and require explicit user confirmation for `truth/edit: ask` paths. If stdout is empty, `sourceOfTruth` is unconfigured — skip silently; add no "not configured" noise.

## Verify at plan time

Before decomposing tasks:

```bash
python3 __GRILL_ADAPTER_ROOT__/scripts/source_truth_settings.py <repo-root> --render-prompt plan-pre
```

During plan review, render the review checklist and apply it to the drafted plan:

```bash
python3 __GRILL_ADAPTER_ROOT__/scripts/source_truth_settings.py <repo-root> --render-prompt plan-review
```

Request revision or user confirmation if the plan explicitly or implicitly edits configured truth paths.

## Relationship to the completion-time Lint

This skill is the **planning-side Verify**. The **execution-side Lint** is a changed-path check that runs during implementation and code-review via `hooks/source-truth-lint.sh` (or by calling `source_truth_settings.py <repo-root> --lint-changed --changed-path <p> --format json` directly). The lint classifies the *actual* touched files: a `block` result must be resolved (revert or route through the upstream process) before the task is complete; an `ask` result needs explicit authorization; `evidence` findings are warnings. The authorization flags (`--authorized-truth-edit`) never bypass `truth/edit: never`.

This is a standalone utility skill, not a host development workflow step; do not auto-invoke the host's completion or verification skills because it ran.
